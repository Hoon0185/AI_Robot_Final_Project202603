# 🤖 Gilbot Robot Simulator (v2.1) 가이드

`simulate_robot.py`는 실제 로봇 하드웨어 없이도 Gilbot 시스템(백엔드 API 및 HMI)의 기능을 테스트할 수 있도록 제작된 가상 로봇 시뮬레이터입니다.

---

## 1. 주요 기능
- **환경별 시간 동기화 (Chrony)**: 실행 모드에 따라 로컬 PC 또는 원격 서버와 시간을 자동으로 맞춥니다.
- **실시간 데이터 보고**: 위치(Odom), 하트비트, 물체 인식 결과(Detection)를 서버로 전송합니다.
- **원격 명령 수신 (Polling)**: HMI에서 내리는 순찰 시작, 비상 정지 등의 명령을 실시간으로 감지하여 동작합니다.
- **상태 시뮬레이션**: 이동 속도 보간, 장애물 감지 대기, 결품 및 오진열 확률 시나리오를 포함합니다.

---

## 2. 시작하기 (실행 방법)

시뮬레이터는 실행 시 위치한 환경에 따라 `local` 또는 `remote` 인자를 사용하여 실행합니다.

### A. 로컬 테스트 모드
로컬 PC에서 API 서버를 함께 실행 중일 때 사용합니다.
```bash
# 기본 실행 (워크스테이션 시간 동기화 시도)
python3 simulate_robot.py local --password [사용자암호]

# 시간 동기화 없이 실행
python3 simulate_robot.py local --no-sync
```

### B. 원격 서버 테스트 모드 (Lightsail)
AWS Lightsail에 서버가 배포되어 있고, 로컬에서 로봇만 시뮬레이션할 때 사용합니다.
```bash
# 서버 시간과 동기화하며 실행
python3 simulate_robot.py remote --password [사용자암호]
```

---

## 3. 명령줄 인자 (Command Line Arguments)

| 인자 | 설명 | 비고 |
| :--- | :--- | :--- |
| `local` | 로컬 API 서버(`localhost:8000`)에 연결 | 기본값 |
| `remote` / `server` | 원격 Lightsail 서버(`16.184.56.119`)에 연결 | |
| `--password [PWD]` | `sudo` 명령 실행 시 필요한 암호 자동 입력 | Chrony 설정에 사용 |
| `--no-sync` | 실행 전 Chrony 시간 동기화 과정을 생략 | 권한 문제 발생 시 사용 |

---

## 4. 인터랙티브 메뉴 안내

시뮬레이터가 실행되면 터미널에서 다음 숫자 키를 입력하여 로봇을 직접 제어할 수 있습니다.

1. **[순찰 시작]**: 서버에서 웨이포인트를 로드하여 순찰 시나리오를 시작합니다.
2. **[비상 정지]**: 현재 이동이나 동작을 즉시 멈추고 `EMERGENCY_STOP` 상태가 됩니다.
3. **[비상 해제/재개]**: 비상 정지된 위치에서 남은 순찰 경로를 다시 시작합니다.
4. **[기지 복귀]**: 모든 동작을 중단하고 원점(0,0)으로 이동합니다.
5. **[메모리 로드]**: 서버의 최신 환경 설정 및 웨이포인트 정보를 다시 불러옵니다.
q. **종료**: 시뮬레이터를 안전하게 종료합니다.

---

## 5. 시뮬레이션 시나리오 상세

### 하트비트 (Heartbeat)
- 로봇은 유휴 상태(IDLE)일 때도 **2초 간격**으로 서버에 현재 위치와 하트비트를 보냅니다.
- 서버(`main.py`)는 마지막 하트비트로부터 **5초 이상** 신호가 없으면 로봇을 `OFFLINE`으로 판정합니다.

### 인식 결과 확률 (Detection logic)
순찰 중 웨이포인트에 도착하면 다음 확률에 따라 인식을 시뮬레이션합니다:
- **정상 (70%)**: 계획된 상품을 정확히 인식합니다.
- **결품 (15%)**: 해당 위치에 상품이 없는 것으로 보고합니다.
- **오진열 (15%)**: 다른 칸의 상품이 놓여 있는 것으로 보고합니다.

### 장애물 감지
- 이동 중 **30%의 확률**로 가상의 장애물을 만납니다.
- 설정된 `avoidance_wait_time`만큼 대기 후 이동을 재개합니다.

---

## 6. 주의 사항
- **권한**: 시간 동기화 기능은 시스템 설정을 변경하므로 `sudo` 권한이 필요합니다. 암호를 입력하지 않거나 `--no-sync`를 사용하면 시간이 맞지 않아 서버에서 로봇을 오프라인으로 잘못 인식할 수 있습니다.
- **네트워크**: `remote` 모드 사용 시 Lightsail 서버의 80포트가 열려 있어야 하며, 로컬 인터넷 연결이 안정적이어야 합니다.
