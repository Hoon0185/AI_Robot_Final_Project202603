# 👁️ Integrated AI Vision System

이 문서는 `v0.0.16` 업데이트에서 도입된 **통합 및 온디맨드 AI 비전 파이프라인**에 대해 설명합니다.

---

## 🏗️ 시스템 아키텍처

기존에 분산되어 있던 YOLO 상품 탐지, QR 바코드 인식, 데이터 검증 로직을 하나의 **`IntegratedPCNode`**로 통합하였습니다.

### 1. 프레임 공유 (Unified Frame)
- 카메라 노드(`/rtsp_image`)로부터 받은 단일 이미지 프레임을 YOLO와 QR 탐지기가 동시에 사용합니다.
- 노드 간 데이터 전송 오버헤드를 제거하여 연산 효율과 동시성을 보장합니다.

### 2. 온디맨드 제어 (On-Demand Activation)
- **트리거**: `patrol_node`가 매대 앞에 도착하여 멈추는 순간 `/ai_mode` 토픽으로 `True`를 발행합니다.
- **연산**: AI 노드는 이 신호를 받았을 때만 분석 루프를 기동합니다.
- **절전**: 분석 완료 후 다시 `False`를 수신하면 루프를 중지하여 CPU 자원을 회수합니다.

---

## 🛠️ 디버깅 및 와치독 (AI Watchdog)

인식이 원활하지 않을 경우, 터미널 로그를 통해 다음과 같은 상태를 실시간으로 확인할 수 있습니다.

### 주요 로그 메시지
- `🔍 [AI Watchdog] 분석 중... (물체: N개 / 바코드: M개)`
  - 현재 로봇이 보고 있는 화면에 상품과 바코드가 각각 몇 개씩 잡히는지 출력합니다.
- `⚠️ [AI Watchdog] 물체는 잡히지만 바코드가 보이지 않습니다.`
  - 이 로그가 뜬다면 YOLO는 상품을 찾았으나 QR 탐지기가 바코드를 못 찾고 있는 상황입니다. (카메라 각도 조절 필요)
- `AI 인식 활성화 (대기 시간: 10.0초 모드)`
  - 현재 로봇이 목적지에서 최대 몇 초간 인식을 대기하는지 보여줍니다.

## 가동 방법

기존 런치 파일에 모두 통합되어 있으므로 별도 실행이 필요하지 않습니다.

```bash
# 전체 시스템 실행 시 자동으로 포함됨
ros2 launch patrol_main total_patrol.launch.py
```

개별 테스트가 필요한 경우:
```bash
ros2 run protect_product integrated_node
```

> [!IMPORTANT]
> 인식이 TIMEOUT이 계속된다면, 로봇의 카메라 각도가 선반의 바코드를 정면으로 향하고 있는지 확인하십시오. 바코드가 인식되어야만 상품 검증 절차가 시작됩니다.
