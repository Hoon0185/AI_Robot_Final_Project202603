# 📦 NPM 실전 가이드 (Gilbot UI 개발용)

길봇 프로젝트의 프론트엔드(React) 개발 시 가장 자주 사용하게 될 핵심 명령어들을 정리한 문서입니다.

---

### **1. 🏗️ 프로젝트 세팅 및 설치**
조종실의 모든 기공물을 설치하거나 새로운 부품(라이브러리)을 가져올 때 사용합니다.

*   `npm install` (또는 `npm i`): 프로젝트에 필요한 모든 라이브러리를 `package.json` 기준으로 한꺼번에 설치합니다. (최초 개발 시작 시 필수!)
*   `npm install <패키지이름>`: 필요한 부품을 하나만 골라서 추가합니다. (예: `npm install axios` - 통신 도구 추가)
*   `npm uninstall <패키지이름>`: 더 이상 필요 없는 부품을 안전하게 제거합니다.

---

### **2. 🚀 개발 모드 가동 (Vite 전용)**
코드를 수정하면 화면에 즉시 반영되는 '개발 조종실'을 여는 명령어입니다.

*   `npm run dev`: 로컬 서버(`http://localhost:5173`)를 실행합니다. **HMR(Hot Module Replacement)** 기능 덕분에 코드를 저장하자마자 화면이 바뀝니다! 👋🎯⚡

---

### **3. 🧱 실전 배포를 위한 빌드 (Build)**
수만 개의 소스 코드를 단 하나의 덩어리로 압축하여 **`Apache`**가 읽을 수 있는 형태로 만드는 매우 중요한 과정입니다.

*   `npm run build`: `dist/` 폴더를 생성하고, 최적화된 HTML/JS/CSS 결과물을 쏟아냅니다. (서버에 올릴 때 필수!)
*   `npm run preview`: 빌드된 결과물이 잘 작동하는지 로컬에서 미리 확인해 보는 기능입니다.

---

### **💡 유용한 팁 (Insight)**

*   **npx vs npm**: 
    - `npm`은 설치된 것을 **'실행'**하거나 **'관리'**할 때 씁니다.
    - `npx`는 설치 없이 일회성으로 도구(예: `create-vite`)를 **'빌려 올 때'** 사용합니다.
*   **node_modules**: 이 폴더는 `npm install`로 받아온 부품 창고입니다. 용량이 매우 크므로 **Git에는 절대로 올리지 않습니다.** (`.gitignore`에 포함 확인!)

---

### **🛠️ 문제 해결 (Troubleshooting)**
*   명령어가 안 먹힐 때: `package.json` 파일이 있는 **`web-ui/`** 폴더 안에서 명령어를 치고 있는지 확인하세요! 👋🎯🚥
