## 1. 시스템 데이터 흐름도
### A. UI에서 장애물 대기 시간을 변경할 때

```text
[웹 UI 슬라이더]
       │
       ▼
[ObstacleInterface] (인터페이스 노드)
       │
       ├─▶ (API 전송) ──▶ [원격 DB 서버] (설정값 영구 저장)
       │
       └─▶ (ROS2 Service: 'obstacle_node/set_parameters')
               │
               ▼
        [ObstacleNode] (장애물 노드)
               │
               └─▶ 'current_wait_time' 파라미터 업데이트 (실시간 반영)
```
### B. 주행 중 전방 장애물을 감지했을 때

```text
[Lidar 센서] ──▶ 토픽: '/scan'
                      │
                      ▼
               [ObstacleNode]
                      │
      ┌───────────────┴───────────────┐
      │ (1단계: 즉시 정지)               │ (2단계: 상태 전파)
      ▼                               ▼
토픽: '/cmd_vel_obstacle'        토픽: '/pause_patrol' (True)
 (속도 0.0 발행)                       │
      │                               ▼
      ▼                         [PatrolNode] (순찰 노드)
 [모터 제어기]                        │
  (로봇 정지)                         └─▶ Nav2 액션 목표 취소 및 대기
```

## 2. 우회 회전 기동 및 수동 조작 프로세스
### A. 신규(고정) 장애물 발생 시 우회 프로세스

```text
[정상 자율 주행 중]
       │
       ▼
[전방 60도 이내 50cm 미만 장애물 최초 감지]
       │
       ├─▶ '/cmd_vel_obstacle' (속도 0.0) 즉시 발행 (로봇 정지)
       ├─▶ '/pause_patrol' (True) 발행 ──▶ PatrolNode 내비 목표 취소
       └─▶ 타이머 시작 (elapsed_seconds 카운트 시작)
               │
               ▼
[지정된 대기 시간(예: 5초) 동안 장애물이 계속 유지됨]
       │
       ├─▶ 'is_blocked' 해제 (False) 및 'is_detouring' 활성화 (True)
       ├─▶ '/pause_patrol' (False) 발행 ──▶ PatrolNode 속도 잠금 해제 요청
       ▼
[우회 기동 개시 (2초간)]
       ├─▶ 전방 20도 구역에 45cm 거리의 '가짜 벽' 발행 ('/scan_virtual')
       ├─▶ Nav2 코스트맵 클리어 서비스 호출 (장애물 잔상 제거)
       └─▶ '/cmd_vel_obstacle'에 '제자리 좌회전(angular.z = 0.7)' 명령 발행
               │
               ▼
[2초 회전 완료 후]
       ├─▶ '가짜 벽' 발행 중단 및 코스트맵 다시 클리어
       └─▶ 정상 주행 복귀 및 가던 목적지로 재출발
```
### B. 일시적 장애물 (사람 등이 비켜난 경우) 프로세스

```text
[장애물 감지로 로봇 정지 및 대기 중]
       │
       ▼
[전방 시야에서 장애물이 사라짐]
       │
       ▼
[사라진 상태가 1.5초(또는 0.5초) 이상 확정 유지됨]
       │
       ├─▶ 'is_blocked' 플래그 해제 (False)
       ├─▶ '/pause_patrol' (False) 발행
       │
       ▼
[PatrolNode]가 신호를 받고 'resend_current_goal()' 실행
       │
       └─▶ 기존에 가려던 목적지 좌표로 다시 출발 (우회 회전 없음)
```
### C. 수동 조작(Teleop) 시 강제 충돌 방지 프로세스** - 테스트 필요

```text
[사용자가 UI나 키보드로 조작 명령 입력] ──▶ 토픽: '/cmd_vel_teleop'
                                            │
                                            ▼
                                     [ObstacleNode]
                                            │
               ┌────────────────────────────┼────────────────────────────┐
               ▼                            ▼                            ▼
      [명령 속도가 0일 때]        [사용자가 '전진' 조작 시]     [사용자가 '후진' 조작 시]
               │                            │                            │
               ▼                            ▼                            ▼
      정지 명령 그대로 통과       [전방 60도 범위 감시]         [후방 60도 범위 감시]
     ('/cmd_vel_obstacle')                  │                            │
                                            ▼                            ▼
                                   [0.5m 이내 장애물 유무?]      [0.3m 이내 장애물 유무?]
                                            │                            │
                            ┌───────────────┴───────┐    ┌───────────────┴───────┐
                            ▼                       ▼    ▼                       ▼
                        <장애물 없음>          <장애물 있음> <장애물 없음>          <장애물 있음>
                            │                       │            │                       │
                            ▼                       ▼            ▼                       ▼
                        조작 명령              선속도(x) 차단    조작 명령              선속도(x) 차단
                        그대로 발행            각속도(z)만 허용  그대로 발행            각속도(z)만 허용
```
