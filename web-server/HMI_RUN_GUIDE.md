# 🚀 Gilbot HMI 실행 가이드

이 문서는 터틀봇(TurtleBot) 로봇 본체에서 HMI(Human-Machine Interface) 키오스크 화면을 실행하는 방법을 안내합니다.

## 1. 사전 준비 (Prerequisites)
- 로봇 전원이 켜져 있고 네트워크(192.168.1.200)에 연결되어 있어야 합니다.
- 로봇에 `xinit` 패키지가 설치되어 있어야 합니다. (설치: `sudo apt install xinit`)

## 2. 로봇 접속 (SSH)
내 컴퓨터의 터미널에서 다음 명령어로 로봇에 접속합니다.
```bash
ssh robot@192.168.1.200
# 비밀번호: robot123 (기본값)
```

## 3. HMI 서비스 실행
로봇 접속 후, `web-server` 디렉토리(또는 스크립트가 있는 경로)로 이동하여 실행 스크립트를 작동시킵니다.
```bash
./start_hmi.sh
```

> [!TIP]
> **자동 디스플레이 감지**: 현재 `start_hmi.sh`는 X 서버가 실행 중이지 않을 경우 자동으로 `xinit`을 통해 디스플레이를 초기화하도록 설정되어 있습니다.

## 4. 스크립트 상세 설명 (`start_hmi.sh`)
이 스크립트는 다음과 같은 안정화 기능을 수행합니다:
- **세션 초기화**: 기존에 실행 중인 크로미움(Chromium) 프로세스와 잠금 파일(`/tmp/chromium-hmi`)을 제거하여 중복 실행 오류를 방지합니다.
- **키오스크 모드**: 주소창과 메뉴바 없이 전체 화면으로 HMI를 띄웁니다.
- **원격 연결**: `http://16.184.56.119:8000/hmi/` 서버를 바라보도록 설정되어 있습니다.

## 5. 문제 해결 (Troubleshooting)

### Q: `xinit: not found` 오류가 발생합니다.
A: 다음 명령어로 패키지를 설치해 주세요:
```bash
sudo apt update && sudo apt install xinit
```

### Q: 화면이 검은색으로만 나옵니다.
A: 로봇의 물리적인 디스플레이(HDMI 또는 전용 LCD)가 연결되어 있는지 확인해 주세요. SSH 환경에서는 브라우저 창이 보이지 않으며, 로봇 본체의 화면에 나타납니다.

### Q: 다른 서버 주소를 보고 싶습니다.
A: `start_hmi.sh` 파일 내의 `TARGET_URL` 변수를 원하는 주소로 수정 후 다시 실행하세요.
```bash
nano start_hmi.sh
# TARGET_URL="http://원하는_주소:8000/hmi/" 로 수정
```

**문의 사항:** 로봇 제어 시스템 및 HMI 관련 문의는 프로젝트 담당자에게 연락 바랍니다.
